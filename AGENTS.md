# Project Instructions

This is the repository for the newest version of the [anmorris.org](https://anmorris.org) website.

## Project Overview

This website is built using **Zola**, a fast Static Site Generator (SSG) written in Rust. It uses **Tera** for templating, **CommonMark** for Markdown files, and TOML for configuration (`zola.toml`).

Packaging, interactive components, and styling are handled using **Vite**, **Tailwind**, **TypeScript**, **Lit**, **xstate**, and **pnpm**.

Code linting and styling is handled by **Prettier** and **ESLint**.

There are currently no unit tests to run.

## Project Structure

- `content/`: All site pages, sections, and blogs. 
- `templates/`: HTML templates using Tera syntax.
- `static/`: Compiled assets, CSS, fonts, and images.
- `zola.toml`: Global site configuration.

## 1. Zola Syntax and Standards

- **Front Matter**: Every markdown file in `content/` must have a valid TOML front matter block enclosed in `+++`, followed by configuration keys (e.g., `title`, `date`, `template`).
- **Internal Links**: Do not use hardcoded paths. Use Zola's internal link mechanism: `@/blog/my-post.md`.
- **Tera Templates**: In `templates/`, use Tera syntax `{{ variable }}` and tags `{% block content %}`. Ensure inheritance is set up correctly for base templates.

## 2. Workflow and Boundaries
- You MAY NOT modify the files in `content/`. The files in `content/` are to never be AI generated or assisted.
- Do not manually edit the `public/` directory; it is auto-generated by `zola build`.
- The theme of this site is fully custom.
- Do not be overly verbose when writing code or markup.
- We are not using SASS.
- Always favor first-party documentation over third-party resources. That means favor the documentation located in this repo and the official docs for Zola, TypeScript, Tailwind, Lit, xstate, and pnpm. If you use a third-party resource, make it clear that you did.
